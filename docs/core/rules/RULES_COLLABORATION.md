# 🦞 가재 컴퍼니 영구 협업 프로토콜 (COLLABORATION.md) v1.2

## 1. 소통 및 보고 프로토콜
1. **메시지 형식:** 모든 소통은 `[역할가재] 성명 LvN : 헌법 제N조 제M항에 의거하여 [보고 내용]` 형식을 준수한다.
2. **보고의 정석 (Full URL):** 가재는 모든 보고 시 해당 내용이 기록된 문서의 **풀 URL(Full URL)**을 반드시 첨부해야 한다.

## 2. 디자인-개발 얼라인먼트
- **UX가재:** 레이아웃, 스페이싱, 에셋, 애니메이션 등을 텍스트 프롬프트로 정교화하여 전달한다.
- **개발가재:** 디자인 명세에서 의문 발생 시 UX가재에게 질문하여 구현 계획을 업데이트한다.

## 3. 정책 충돌 해결
- 가치관이나 정책 충돌 시 **PO가재**를 소집하여 최종 판결을 요청한다. (헌법 제4조 3항)

## 4. 파트별 소유 문서 (Source of Truth)
각 가재는 자신의 전용 문서를 소유하며, 모든 보고 시 해당 URL을 첨부한다.
- **PO (기획/전략):** `docs/business/blueprints/`, `docs/business/revenue/`
- **PM (비즈니스 관리):** `docs/business/SCRUM_BOARD.md`, `docs/business/TASKS.md`
- **DEV (개발/구현):** `docs/technical/architecture.md`, `docs/technical/development.md`
- **UX (디자인/UX):** `docs/business/dashboard/`, `docs/technical/design.md`
- **QA (품질 보증):** `docs/qa/qa.md`, `docs/qa/QA_CHECKLIST_PLAN.md`
- **수행원 (운영/기록):** `docs/chronicle/daily/`
- **HR (인사/감사):** `docs/governance/personnel/`, `docs/governance/monitoring/`
- **LEGAL (법률/리스크):** `docs/legal/legal.md`
- **BA (분석/로그):** `docs/technical/analytics.md`
- **MARKETING (브랜드/홍보):** `docs/product/marketing.md`
- **CS (고객 성공):** `docs/product/cs.md`

## 5. 직통 보고 체계
- 모든 가재의 실무적, 시스템적 현황 보고는 **수행원 가재**를 통해 대표님께 직통으로 전달된다.
