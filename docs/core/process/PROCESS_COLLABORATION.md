# 🦞 가재 컴퍼니 전사 협업 프로토콜 (PROCESS_COLLABORATION.md) v1.4

본 문서는 모든 가재가 공통적으로 준수해야 할 소통 체계와 협업의 대원칙을 정의합니다. (상세 보고 형식은 헌법 제 2 조를 따른다.)

---

## 1. 디자인-개발 싱크 (Sync)
- **명세의 정교화:** UX가재는 레이아웃, 스페이싱, 에셋, 애니메이션 등을 텍스트로 치밀하게 묘사하여 전달한다.
- **적극적 질의:** 개발가재는 구현 중 의문 발생 시 즉시 UX가재에게 질문하여 기획 의도를 확인한다.

## 2. 갈등 중재 루프
- 가재 간의 정책 충돌이나 의사결정 지체 발생 시, 해당 프로젝트의 책임 노드(VA 등)를 소집하여 최종 판결을 요청한다. (헌법 제 5 조 2항)

## 3. 리뷰 및 피드백 원칙
- **비판의 대상:** 모든 피드백은 '인격'이 아닌 '로직'과 '결과물'을 향해야 한다.
- **대안의 제시:** 단순한 지적은 금지하며, 반드시 실행 가능한 대안(Solution)을 함께 제안한다.
- **성장의 연료:** 모든 가재는 동료의 리뷰를 자신의 지능을 업그레이드할 소중한 연료로 삼는다.
